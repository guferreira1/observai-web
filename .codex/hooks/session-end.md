# Session end checklist

Before finishing work:

1. Review changed files.
2. Confirm no secrets were added.
3. Confirm no frontend public env variable contains sensitive data.
4. Confirm components remain small and focused.
5. Confirm raw fetch is not scattered inside visual components.
6. Confirm server state is not stored in global client stores.
7. Confirm loading, empty, error and success states were considered.
8. Confirm accessibility basics were considered.
9. Run available validation commands when possible.
10. Update `.codex/memory.md` for meaningful sessions.

Handoff must include:

- what changed
- validation performed
- risks
- pending items
