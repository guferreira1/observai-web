# Session start checklist

Before starting work:

1. Read `AGENTS.md`.
2. Read `.codex/README.md`.
3. Read `.codex/rules/project-decisions.md`.
4. Read the rules related to the task.
5. Read the relevant agent profile.
6. Read `.codex/memory.md`.
7. Confirm the current branch is not `main`.
8. Identify the affected feature.
9. Identify whether the work needs UI, API, state, schema, mapper or test changes.
10. Identify validation commands available in the project.

Do not start implementation before understanding the feature boundary and expected user behavior.
