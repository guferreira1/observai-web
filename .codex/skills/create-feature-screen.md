# Skill: create feature screen

Use this workflow when creating or changing a product screen.

## Steps

1. Identify the feature folder.
2. Define the user goal for the screen.
3. Define loading, empty, error and success states.
4. Define required API data.
5. Create or reuse typed API client functions.
6. Create schemas and mappers when needed.
7. Create feature container for data and state orchestration.
8. Create presentational components.
9. Keep JSX readable and split large sections.
10. Add tests for mappers, rules, forms or complex components.
11. Review accessibility.
12. Review performance and client component boundaries.

## Output expectation

A screen should be clear, typed, accessible, testable and aligned with the ObservAI product language.
